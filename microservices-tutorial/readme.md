# 微服务系统学习需求说明（面向 Java 开发）

## 1. 学习背景

- 当前已具备 Java 开发经验，希望系统掌握微服务相关知识。
- 工作或未来方向中，预计会接触或主导基于 Spring 生态的微服务项目。
- 目标不仅是“会用框架”，而是理解微服务的架构思想、设计原则、常见问题与解决方案。

## 2. 目标读者与前置技能

### 2.1 目标读者画像

- 至少 1–2 年 Java 开发生产经验。
- 使用过 Spring / Spring Boot 开发 Web 应用。
- 希望：
  - 能从零设计一个简单的微服务系统；
  - 能读懂并维护现有 Spring Cloud 微服务系统；
  - 能参与到微服务架构的技术决策中。

### 2.2 前置技能

学习前建议具备的知识基础：

- Java 语言基础（集合、并发、JVM 基础、异常处理等）。
- Web 开发基础：HTTP 协议、REST 风格、常见状态码。
- Spring / Spring Boot 基础：
  - IOC / AOP 基本概念；
  - Controller、Service、Repository 的分层结构；
  - Spring Boot 自动配置和启动流程的基本认识。
- 基本的数据库与 SQL 经验（MySQL 或其它主流 RDBMS）。
- 基本的 Git 使用经验。

如上述部分技能暂不熟练，可在正式开始前用 1–2 周时间补课。

## 3. 学习目标

通过本学习计划，最终希望达成以下目标：

1. **理解微服务架构思想**
   - 明白为什么从单体走向微服务，以及微服务适用场景与限制。
   - 掌握服务拆分、边界划分的基本方法（DDD 概念层面即可）。

2. **掌握 Spring Cloud / Spring Cloud Alibaba 栈的核心组件**
   - 服务注册与发现（Eureka / Nacos）。
   - 负载均衡与服务调用（OpenFeign / Spring Cloud LoadBalancer）。
   - 配置中心（Spring Cloud Config / Nacos Config）。
   - 网关（Spring Cloud Gateway 或 Nginx + 网关方案）。
   - 熔断、限流、降级（Resilience4j / Sentinel）。

3. **掌握服务间通信与事务一致性解决方案**
   - REST/RPC 通信模式；
   - 同步调用与异步消息（Kafka/RabbitMQ）；
   - 分布式事务常见模式：事务消息、补偿事务、最终一致性思路。

4. **掌握微服务可观测性与稳定性方案**
   - 日志采集与链路追踪（Spring Cloud Sleuth / SkyWalking / Zipkin 等思路）。
   - 指标与监控（Micrometer / Prometheus / Grafana 思路）。
   - 健康检查、服务熔断与自动恢复的基础概念。

5. **掌握容器化与部署基本能力**
   - 编写 Dockerfile，将 Spring Boot 应用容器化。
   - 使用 Docker Compose 搭建本地多服务环境。
   - 理解 Kubernetes 基本资源概念与部署方式（了解或入门级别）。

6. **具备从零实现一个小型微服务项目的能力**
   - 至少完成一个包含 3–5 个服务的实践项目：
     - 用户服务、订单服务、商品服务等；
     - 包含服务发现、网关、配置中心、监控等基础设施；
     - 支持基本的 CI/CD（如基于 GitLab CI/GitHub Actions 的简单流水线）。

## 4. 学习范围与模块划分

计划覆盖以下知识模块：

1. **微服务基础与架构概念**
   - 单体 vs 分布式 vs 微服务；
   - CAP、BASE、最终一致性等分布式基础；
   - DDD 简要概念与服务边界设计。

2. **Spring Boot 与服务拆分实践**
   - 通过一个单体应用示例，拆分为多个独立服务；
   - 服务拆分时的数据划分、接口设计、依赖处理。

3. **服务治理基础设施**
   - 注册中心：Eureka / Nacos；
   - 配置中心：Spring Cloud Config / Nacos Config；
   - 网关：Spring Cloud Gateway 的路由、过滤器、鉴权；
   - 服务调用：OpenFeign，负载均衡与超时重试配置。

4. **服务可靠性与容错**
   - 熔断、隔离、限流、降级；
   - 常见工具：Resilience4j 或 Sentinel；
   - 超时控制、重试策略、舱壁隔离等设计。

5. **消息中间件与异步架构**
   - 为什么要用消息队列；
   - 使用 Kafka / RabbitMQ 进行解耦与削峰；
   - 以订单创建为例的异步处理与最终一致性。

6. **安全与认证授权**
   - 统一认证、单点登录（SSO）思路；
   - JWT/OAuth2/OpenID Connect 基本概念；
   - 在网关层做认证与授权的常见模式。

7. **可观测性：日志、监控与链路追踪**
   - 统一日志规范（traceId、spanId、结构化日志）；
   - Spring Cloud Sleuth 链路追踪；
   - 指标上报与监控告警的整体方案。

8. **容器化与部署**
   - Docker 基本概念与常用命令；
   - 为各服务编写 Dockerfile；
   - 使用 Docker Compose 编排本地多服务集群；
   - 了解 Kubernetes 基本概念（Deployment、Service、Ingress 等）。

9. **CI/CD 与工程化实践**
   - 基于 Git 的分支策略与代码管理；
   - 使用 GitHub Actions / GitLab CI 配置基础流水线：
     - 构建、测试、打包镜像、部署；
   - 环境分层：开发、测试、预发布、生产的配置与部署差异。

## 5. 学习方式与输出要求

### 5.1 学习方式

- **理论 + 实践结合**：每个模块都配套小练习或小项目。
- **循序渐进**：从单体到拆分，再逐步引入服务治理、可观测等能力。
- **项目驱动**：以电商/订单/博客等业务为核心贯穿全程。

### 5.2 输出要求

每个阶段需要形成可验证的输出：

- 阅读笔记：个人总结、关键概念理解，建议以 Markdown 记录。
- Demo 项目代码：上传到 Git 仓库（区分各阶段分支或目录）。
- 小结文档：每一大模块完成后输出阶段总结（踩坑、经验、TODO）。

最终成果：

- 1 个完整的微服务示例项目源码；
- 一套完整的学习笔记与架构设计说明；
- 一份可复用的微服务实践 Checklist（可用于以后项目立项或评审）。

## 6. 学习时间与节奏建议

- 预估总投入：**6–10 周**（取决于每天可投入时间）。
- 建议节奏：
  - 每周 4–6 小时学习时间（工作日 + 周末结合）；
  - 每周至少 1 次集中编码实践（2–3 小时）。

实际计划安排将体现在 `plan.md` 中，可根据工作与生活节奏灵活调整。

## 7. 学习环境与工具约束

- 开发语言：Java 8+（推荐 11 或以上）。
- 主框架：Spring Boot 2.x/3.x，Spring Cloud 相关组件。
- 开发工具：IntelliJ IDEA 或 Eclipse（推荐 IDEA）。
- 构建工具：Maven 或 Gradle（建议 Maven，生态资料更多）。
- 版本控制：Git（GitHub/GitLab/Codeup 等均可）。
- 容器平台：Docker（本地环境必须可用）。

## 8. 评估方式

建议从以下几个维度自评学习效果：

1. **概念理解**  
   能清晰解释微服务相关关键概念，并能对比多种方案。

2. **实践能力**  
   能独立创建并运行一个包含 3–5 个服务的微服务系统，包括配置中心、注册中心、网关等。

3. **问题排查能力**  
   面对调用超时、熔断、配置错误、服务下线等问题，能快速定位并解决。

4. **架构评审能力**  
   能对一个微服务架构给出基本评估（例如服务划分是否合理、是否过度复杂、基础设施是否缺失等）。

根据以上评估结果，可在后续迭代中调整学习重点（如加强容错、加强可观测性等）。
