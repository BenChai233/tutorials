## 并发编程学习需求说明

本仓库用于通过 Codex 辅助的自学，系统掌握 Java 并发编程。文档清晰列出学习目标、前置条件、环境要求、实践产出和进阶路线，配合 `plan.md` 的时间表即可直接开学。

### 学习目标
- 理解并发的动机、线程模型与内存模型（Java Memory Model）。
- 熟练使用 Java 基础并发工具（`Thread`、`Runnable`、`synchronized`、`volatile`、`Lock`、`Condition`、`Atomic`、`Future`、`Executor`）。
- 能够分析并解决常见并发问题（竞态、可见性、死锁、活锁、伪共享、阻塞/饥饿）。
- 能设计并实现健壮的并发组件（线程安全集合、生产者-消费者、异步任务编排）。
- 会用性能诊断与测试手段验证并发代码的正确性与效率。

### 适用人群
- Java 后端/中间件/基础设施开发。
- 有 1 年以上 Java 经验，想系统补齐并发理论与实践。

### 前置条件
- 熟悉 Java 语法、集合、异常、泛型。
- 具备基础的面向对象设计能力。
- 能运行命令行工具，使用 Git 与构建工具（Maven/Gradle）。

### 开发与实验环境
- JDK 17+（最好同一大版本以避免行为差异）。
- 构建工具：Maven 或 Gradle 均可，选其一即可。
- IDE：IntelliJ IDEA 或 VS Code（均支持调试与线程视图）。
- 工具：`jconsole`、`jvisualvm` 或 `jmc`，用于线程与性能观测；`jcmd`/`jstack`/`jmap` 用于诊断。
- 本仓库结构：按章节放置示例代码与实验记录（建议 `chapter-XX-*` 目录）。

### 学习方式（结合 Codex）
- 先阅读 `plan.md` 对应章节，明确概念与目标。
- 按章节写出最小可运行示例，小步递进；用 Codex 辅助解释栈轨迹、并发 bug、或生成 scaffold，但核心逻辑尽量手写。
- 每个专题增加对照实验：先写有问题的版本，再写正确或更优的版本，对比日志/性能/正确性。
- 对每个练习写下「现象、原因、修正、总结」四要素，积累故障手册。

### 里程碑产出
- 每章：示例代码、实验记录（问题与结论）、关键笔记。
- 阶段性：可复用的并发工具函数/组件（如阻塞队列、任务编排器）。
- 终局：一份并发最佳实践清单和性能调优 checklist。

### 推荐资料
- 《Java 并发编程实战》（JCIP）
- 《Java 并发编程的艺术》
- Java 官方并发包文档 (`java.util.concurrent`)
- JSR-133: Java Memory Model 规范概述
- 相关博客与讲座：Martin Kleppmann、Doug Lea 的论文/邮件列表归档

### 如何开始
1) 安装 JDK 17+，验证 `java -version` 和 `javac -version`。  
2) 选择 Maven 或 Gradle，创建基础工程或使用本仓库目录逐章添加示例。  
3) 阅读 `plan.md`，按周执行任务与练习，记录实验与心得。  
4) 每完成一章，复盘：能否口述模型、独立实现练习、写出对照测试。  
5) 完成全部计划后，整理自己的并发实践手册并回灌到仓库。
